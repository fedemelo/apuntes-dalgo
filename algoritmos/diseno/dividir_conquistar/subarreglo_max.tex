\subsection{Subarreglo máximo}

Un \concept{subarreglo máximo} es un pedazo contiguo no vacío de un arreglo de números cuya suma (la suma de sus elementos) es mayor que la de cualquier otro subarreglo contiguo. El problema de encontrar un subarreglo máximo se define como:
\begin{itemize}
  \item \textbf{Entrada}: arreglo con \(n\) números: \([a_1, a_2,\ldots,a_n]\).
  \item \textbf{Salida}: subarreglo contiguo \([a_i, a_{i+1},\ldots,a_j]\) no vacío tal que la suma \(\sum_{k=i}^j a_k\) es máxima.
\end{itemize}
\say{es máxima} se puede formalizar en una expresión matemática que básicamente dice que esa sumatoria es mayor que cualquier otra sumatoria de elementos contiguos que se pueda formular. Nótese que podría haber más de un subarreglo máximo: podrían haber varios cuya suma sea igual. Basta con hallar cualquiera que sea máximo.

Hay dos familias de instancias de este problema que son muy fáciles:
\begin{itemize}
  \item Cuando todos los valores del arreglo son positivos, la respuesta es todo el arreglo, porque todos los valores aportan a una mayor suma.
  \item Cuando todos los valores son negativos, se quiere que la suma sea lo menos negativa, por lo que se forma el subarreglo con un solo elemento, el menos negativo.
\end{itemize}
Vale la pena entonces pensar en instancias con valores tanto positivos como negativos, que son menos triviales.

Antes de iniciar la implementación, se transforman levemente las especificaciones del problema por unas equivalentes:
\begin{itemize}
  \item \textbf{Entrada}: arreglo con \(n\) números: \([a_0, a_2,\ldots,a_{n-1}]\).
  \item \textbf{Salida}: índices \(i\), \(j\) del arreglo, donde \(0 \leq i \leq j < n\) tales que la suma \(\sum_{k=i}^j a_k\) es máxima.
\end{itemize}
Los cambios son dos: primero, se utilizan índices basados en 0 para el arreglo (menos usual en notación matemática, pero más compatible con lenguajes de programación) y segundo, ahora se buscan los índices que delimitan el máximo subarreglo, en lugar del subarreglo como tal.

\subsubsection{Solución por fuerza bruta}

Una primera solución a este problema se consigue simplemente comparando todas las posibilidades de subarreglos y eligiendo el que tiene la mayor suma:
\begin{pseudocode}
def max_subarray_brute_force(nums: arreglo[int]) -> tuple[int, int, int]:
    largest = |\(-\infty\)|

    best_i = 0
    best_j = 0

    for i=0 to len(nums)-1:
        total = 0
        for j=i to len(nums)-1:
            total += nums[j]
            if total > largest:
                largest = total
                best_i = i
                best_j = j
    return best_i, best_j, largest
\end{pseudocode}
Ese algoritmo cuenta con dos ciclos anidados que recorren el arreglo, por lo que intuitivamente su complejidad temporal es \(O(n^2)\).

\subsubsection{Solución por dividir y conquistar}

Para diseñar un algoritmo usando dividir y conquistar, el primer paso convencional es partir el arreglo. Con eso, cada subproblema consiste en hallar un subarreglo máximo dentro del respectivo pedazo. El cuidado que hay que tener con esa solución es que, al partir el arreglo, puede pasar que se partió justo donde estaba el subarreglo máximo. Debemos cerciorarnos de contemplar esa posibilidad en el paso de combinar las soluciones. Con eso, las etapas son:
\begin{enumerate}
    \item Dividir el arreglo en dos mitades.
    \item Conquistar: Para cada mitad, hallar un subarreglo máximo recursivamente.
    \item Combinar: Comparar los subarreglos máximos de cada mitad para elegir el mayor y también compararlos con los subarreglos que pasan por donde se dividió el arreglo.
\end{enumerate}

Igual que antes, nos preocupamos primero de dividir y conquistar y procrastinamos el combinar encapsulándolo en una función \inline{max_crossing_subarray}. Podemos escribir el seudocódigo para las primeras dos etapas:
\begin{pseudocode}
def max_subarray_divide_conquer(nums: arreglo[int], i: int, j: int) -> tuple[int, int, int]:
    if i == j: return i, j, nums[i]

    k = i + (j - i) // 2

    left_i, left_j, left_sum = max_subarray_divide_conquer(nums, i, k)
    right_i, right_j, right_sum = max_subarray_divide_conquer(nums, k+1, j)
    cross_i, cross_j, cross_sum = max_crossing_subarray(nums, i, k, j)

    if left_sum >= right_sum and left_sum >= cross_sum:
        return left_i, left_j, left_sum

    if right_sum > cross_sum:
        return right_i, right_j, right_sum

    return cross_i, cross_j, cross_sum
\end{pseudocode}

Para maximizar la suma, queremos maximizar la de la izquierda y también maximizar la de la derecha. En este caso, usar un algoritmo exhaustivo no es ingenuo: se toma el subarreglo máximo de la izquierda, el máximo de la derecha, y se juntan:
\begin{pseudocode}
def max_crossing_subarray(nums: arreglo[int], i: int, k: int, j: int) -> tuple[int, int, int]:
    left_sum = 0
    largest_sum_left = |\(-\infty\)|
    for l=k-1 downto i:
        left_sum += nums[l]
        if left_sum > largest_sum_left:
            largest_sum_left = left_sum
            best_l = l

    right_sum = 0
    largest_sum_right = |\(-\infty\)|
    for r=k to j:
        right_sum += nums[r]
        if right_sum > largest_sum_right:
            largest_sum_right = right_sum
            best_r = r

    return best_l, best_r, largest_sum_left + largest_sum_right
\end{pseudocode}
