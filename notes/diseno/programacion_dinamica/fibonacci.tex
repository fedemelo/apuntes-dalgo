\subsection{Sucesión de Fibonacci}

La sucesión de Fibonacci comienza con 0 y 1, y cada número posterior es la suma de los dos anteriores. Formalmente:
\begin{align*}
    F(0) &= 0 \\
    F(1) &= 1 \\
    F(n) &= F(n-1) + F(n-2) \quad \text{para } n > 1
\end{align*}
El problema de calcular el enésimo número de Fibonacci cumple con las características para resolverse eficientemente usando programación dinámica: 
\begin{itemize}
    \item Se puede interpretar como un problema de optimización si el objetivo es encontrar la mínima cantidad de sumas para calcular $F(n)$.
    \item Tiene subestructura óptima: la solución óptima para $F(n)$ se construye a partir de las soluciones óptimas para $F(n-1)$ y $F(n-2)$.
    \item Presenta subproblemas superpuestos: para calcular $F(n)$ se necesitan calcular $F(n-1)$ y $F(n-2)$; para calcular $F(n-1)$ se necesitan calcular $F(n-2)$ y $F(n-3)$. En tan solo dos pasos, ya se ha calculado $F(n-2)$ dos veces. A medida que se avanza, la cantidad de cálculos repetidos crece exponencialmente.
\end{itemize}

\subsubsection{Algoritmo recursivo}

El algoritmo recursivo, que es el más intuitivo, es una traducción directa de la definición matemática:
\begin{pseudocode}
función fibonacci(n: entero) -> entero
    if n == 0 return 0
    if n == 1 return 1

    return fibonacci(n-1) + fibonacci(n-2)
\end{pseudocode}

Su ineficiencia radica precisamente en que presenta subproblemas superpuestos. 

\subsubsection{Programación dinámica top-down}

Una primera optimización es incluir memoización en la implementación recursiva para eliminar la redundancia que introducen los problemas superpuestos:
\begin{pseudocode}
función fibonacci_top_down(n: entero) entero
    soluciones = |\textrm{arreglo de tamaño }$n+1$|
    |\br{cbi}|soluciones[0] = 0
    soluciones[1] = 1|\br{cbf}|

    |\br{lli}|for i=2 to n
        soluciones[i] = -1|\br{llf}|

    return fibonacci_top_down_recursion(n, soluciones)

función fibonacci_top_down_recursion(n: entero, soluciones: arreglo) entero
    if soluciones[n] >= 0 
        return soluciones[n]

    soluciones[n] = fibonacci_top_down(n-1, soluciones) + fibonacci_top_down(n-2, soluciones)
    return soluciones[n]
\end{pseudocode}
\begin{annotations}
    \codebrace{cbi}{cbf}{Se añaden los casos base directamente al arrreglo}
    \codebrace[width=9cm]{lli}{llf}{Se llena el resto del arreglo con -1, que indica que no se ha calculado ese término de la secuencia. Es inambiguo porque los términos calculados serán positivos}
\end{annotations}

Esa implementación es top-down. Evidentemente, para calcular el enésimo número de Fibonacci es necesario calcular todos los anteriores, por ende, el algoritmo top-down es menos eficiente que el algoritmo bottom-up: ambos calculan todos los términos anteriores, pero el top-down tiene el sobrecosto de la recursión.

\subsubsection{Programación dinámica bottom-up}

El método bottom-up resuelve primero los problemas más pequeños. En este caso, eso traduce naturalmente a calcular primero los menores términos de la sucesión, o lo que es lo mismo, a calcularlos en orden:
\begin{pseudocode}
función fibonacci(n: entero) entero
    soluciones = |\textrm{arreglo de tamaño }$n+1$|
    soluciones[0] = 0
    soluciones[1] = 1
    
    for i=2 to n:
        soluciones[i] = soluciones[i-1] + soluciones[i-2]

    return soluciones[n]
\end{pseudocode}
Este problema tiene la particularidad de que no hace falta preservar las soluciones previas, de forma que se puede ahorrar el espacio en memoria que ocupa el arreglo de soluciones:
\begin{pseudocode}
función fibonacci(n: entero) entero
    solucion_trasanterior = 0
    solucion_anterior = 1
    
    for i=2 to n:
        solucion = solucion_anterior + solucion_trasanterior
        solucion_trasanterior = solucion_anterior
        solucion_anterior = solucion

    return solucion
\end{pseudocode}
Se obtiene un algoritmo cuya complejidad temporal es $\Theta(n)$ y cuya complejidad espacial es $\Theta(1)$.
